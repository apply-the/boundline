# Roadmap

Welcome to the official **Boundline** Roadmap. Here we track the evolution of our *Bounded Cognitive Runtime*.

::: info Vision
The goal of Boundline is to transform non-deterministic AI iterations into predictable, traceable, and governable software delivery processes.
:::

## Recently Delivered

- **`076` — Recursive Stage Refinement Profiles** (`0.76.0`): Bounded,
  inspectable stage-refinement loops over session-native runtime with
  `planner → critic → planner → finalizer` pattern, structured round packets,
  hard round limits, no-progress detection, time budgets, and 9 closed stop
  reasons. CLI flags `--refine`, `--no-refine`, `--max-rounds N` with status,
  next, and inspect integration.
- **`075` — Adaptive Governance Calibration** (`0.75.0`): Graduated control
  levels (advisory → catch → rule → hook) with calibration policy,
  `boundline override` for operator bypass, and trust-evolution tracking.
- **`074` — Review Councils & Role-Gated Governance** (`0.74.0`): Guardian
  activation router, council adjudication, and role-gated governance with
  trace-visible override and escalation records.
- **`073` — Contextual Help & Documentation Architecture**: `boundline help-next`
  operator guidance surface with Canon companion spec.
- **`072` — Evals & Runtime Observability** (`0.73.0`): Local quality evals,
  trace compaction with five retention classes, structured event vocabulary,
  JSONL export, runtime metrics, and CI-compatible eval summaries.
- **`071` — Capability Provider Protocol** (`0.72.0`): First native provider
  contract with explicit registration, fail-closed activation, permission
  admission, and bounded execute-plus-evidence collection.
- **`070` — Large Codebase Context Substrate** (`0.71.0`): Local context
  substrate with fidelity tiers, omission findings, repository-map readiness,
  digest-backed compaction, patch-safe edit guards, and snapshot-cache
  boundaries.
- **`069` — Plan Analysis Contract** (`0.70.0`): Read-only planning-coherence
  gate that validates cross-artifact consistency, source-attributed findings,
  and withholds execution handoff on contradictions or producer gaps.
- **`068` — Backlog Contract** (`0.69.0`): First formal backlog-quality gate
  blocking closure-limited Canon backlog packets, requesting clarification, and
  projecting additive backlog-quality state.

## <i class="fa-solid fa-rocket" style="color: #38c7ff;"></i> Upcoming Features & Topics

### Execution & Safety
- **Sandboxed Execution & Secret Inheritance** (seed `13`): Local safety
  boundaries, path/network policies, and read-only/dry-run mutation modes.
- **Completion Verification Runtime** (seed `18`): Fresh-proof gate before task
  or stage completion.
- **Plan Execution Orchestration** (seed `19`): Sequential execution control
  plane with checkpoint and resume.

### Provider & Gateway
- **AI Gateway & Inference Economics** (seed `14`): Route health, local vs
  remote transmission decisions, route budgets, and latency telemetry.
- **Browser & Visual Testing Provider** (seed `15`): Provider via protocol.
- **Experimental RecursiveMAS Provider Adapter** (seed `17`): External
  latent-space provider research track.

### Memory & Knowledge
- **Session Memory & Repository Knowledge Distillation** (seed `16`): Memory
  hygiene and cross-session knowledge persistence.
