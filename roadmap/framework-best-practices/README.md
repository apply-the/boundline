# Framework Best Practices

Questa raccolta contiene linee guida pratiche per framework frontend, backend, mobile e full-stack.

L’obiettivo non è descrivere “come si usa” ogni framework, ma definire regole utili in code review e design review:

- struttura dei progetti
- separazione responsabilità
- dependency injection
- gestione stato
- gestione errori
- testabilità
- performance
- sicurezza
- logging e osservabilità
- anti-pattern ricorrenti
