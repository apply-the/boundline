# Boundline Guidance Catalog — Phase 6

This package extends the Boundline Guidance Catalog with optional ecosystem guidance.

Included:

- Modern frontend frameworks guidance
- Rails and Laravel guidance
- Mobile application guidance
- Data and AI systems guidance
- Optional ecosystem guardian rule seeds
- Phase 6 manifest

This package is designed for S2.1 Guidance And Guardian Capabilities.

## Relationship To Previous Phases

Previous phases covered:

- cross-cutting clean code, architecture, and testing
- major backend/general-purpose languages
- service frameworks
- operations, resilience, and supply chain

Phase 6 adds optional ecosystem coverage that should remain pack-driven rather than core-mandatory:

- Angular, Vue, Svelte, Next.js-style frontend concerns
- Rails and Laravel applications
- native/mobile applications
- data pipelines and AI/ML systems

## Canon Awareness

These files may be used as:

- optional Boundline built-ins
- shared expert pack content
- workspace overrides
- Canon-governed standards after promotion

Framework preferences in this package are recommendation-level by default. They become mandatory only when promoted by workspace policy, Canon-governed standards, or explicit expert pack configuration.
