# Boundline Guidance Catalog — Phase 1

This package contains the first operational content layer for S2.1 Guidance And Guardian Capabilities.

It is not a replacement for S2.1. S2.1 defines how Boundline loads, resolves, executes, traces, and exposes guidance and guardians.

This package defines the initial content catalog that S2.1 can consume:

- Clean Code guidance
- Architecture guidance
- Testing guidance
- Rust guidance
- TypeScript/Node guidance
- Guardian rule seeds
- A follow-up specification for catalog governance and packaging

## Intended Use

Use this package as input for a new Boundline specification:

`Guidance Catalog And Guardian Rule Packs`

The goal is to convert engineering best practices into structured, versioned, inspectable guidance and guardian capabilities.

## Relationship To Canon

Canon may publish governed versions of these guidance files as project memory or governed standards.

Boundline may also consume local workspace overrides, shared expert packs, and built-in guidance.

Canon is the highest external governed authority, but Boundline must remain Canon-aware rather than Canon-dependent.
